~b Per-stream drift capture on /tick

The platform now records which streams drift on every 5-min tick. Walking the snapshot timeline yields per-stream "first point of diff" -- no re-baseline, no manual sweep. Deployed to the live cadence Cloud Run service.

Shipped · Deployed

What changed

The cadence /tick endpoint already computed per-stream verdicts (SweepResult.PerStream) but only persisted aggregate counts. Now it also embeds a drifts array on each snapshot doc -- every non-MATCH stream this tick:

"drifts": [ { "stream": "ETHER.ORDERS", "verdict": "Differ", "prodFp": -439..., "cloneFp": 405..., "bizCols": 31 }, ... ], "driftSchemaVersion": 1

Design call: embedded array, not a subcollection

Operator asked for a "per-stream subcollection." Built it as an embedded drifts array on the snapshot doc instead. Same capability, simpler + cheaper:

Embedded array (chosen)Subcollection
Writes / tick1up to 208 (batched)
Read for timeline walkfree with the snapshotextra fetch per snapshot
/reset/wipe cleanupremoved with parent docorphaned -- Firestore does not cascade-delete subcollections
Doc size~tiny (only non-MATCH; ~25 entries typical, <<1MiB)n/a
collectionGroup query across snapshotsno (read docs + filter in code)yes

Only the non-MATCH streams are stored (Differ/Error/Skip). Absence from drifts = the stream matched. The aggregate counts give the totals. Compact and sufficient for first-diff analysis. If collectionGroup queryability is ever needed, switching to a subcollection is localized to the write block.

Verified live

$ POST /tick -> sweep-20260528T062723 match=183/208 (88%) differ=25 $ GET /reset/status -> lastSweep.drifts: present: True count: 25 ETHER.BARISTA_LOGINS Differ prodFp=-5243911578550057966 cloneFp=2846889560218871543 ETHER.CART_PRODUCTS Differ prodFp=3634625167678047518 cloneFp=-4239249006143337441 ETHER.CARTS Differ ... (all 25 ETHER)
What this unlocks. Going forward, every tick records exactly which streams differ. To find a stream's first point of diff: walk the snapshot timeline oldest->newest, the earliest snapshot it appears in drifts with verdict=Differ is its first divergence. No re-baseline, no manual Driver sweep -- the autonomous service captures it.
Note on the current 25. The ETHER streams already differing today first diverged (aggregate) at 03:31 UTC -- before this capture existed, so their individual first-diff ticks weren't recorded. From this deploy onward, any stream that flips MATCH->DIFFER is captured with its precise tick. After the next re-clone, the full per-stream first-diff timeline accumulates from zero.
