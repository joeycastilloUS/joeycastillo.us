1. Perf -- 142s to 3.3s

ClassifiedSweep now runs one combined query per DB -- HASH_AGG(bizcols) + COUNT(*) + MAX(timestamp cursor) in a single UNION ALL. The MAX piggybacks the full scan the fingerprint already does, so lag + row delta cost nothing extra. INFORMATION_SCHEMA metadata (biz cols both DBs + column types) is cached 60 min. This eliminated the two separate stats passes and the triple metadata reads.

Verdicts identical to Board 16 -- pure speedup. The autonomous 5-min cadence keeps warehouse + cache warm, so ticks run ~3s; the cold ~230s cost happens only on a cold start or after the 60-min cache TTL.

2. Acks -- accept a legit diff

cadence-acks Firestore collection + endpoints GET /acks, POST /ack?stream=&reason=&by=, POST /ack/clear?stream=. /tick loads acks and the classifier downgrades a would-be Signal to Benign with acknowledged: <reason>. Go.Ack launcher: list / "<stream>" "<reason>" / clear "<stream>".

Verified live. ETHER.USERS had aged past the 6h tolerance on the frozen clone -> Signal. Go.Ack "ETHER.USERS" "frozen clone during migration -- expected" -> next tick: signal=0, ETHER.USERS reads Benign -- acknowledged: frozen clone during migration -- expected. Then cleared the ack. Full lifecycle works. This is the mechanism for accepting new-Airbyte legitimate differences post-cutover.

3. Escalation -- the Signal worklist

GET /escalate returns the latest snapshot's Signal streams (the worklist a human or the deep engine should investigate) + ParityL3Url + autoEscalate=false.

Honest scope: the auto-call to airbyte-parity /parity/l3 (deep column-signal / row-delta) is contract-pending -- it needs the engine's request/response shape confirmed + a cadence-svc -> airbyte-parity invoker IAM grant. Rather than ship a fragile guess against an unverified contract (and with 0 standing Signals once acked), the worklist endpoint is the integration point; flip autoEscalate on once the call is wired.

4. Retention

POST /reset/retention?days=N (default 14, X-Reset-Confirm: yes guarded) prunes cadence-snapshots older than the cutoff in 500-doc batches. Deployed + guarded; not fired -- didn't want to delete the timeline you're actively watching. Run it when the snapshot count needs bounding.
