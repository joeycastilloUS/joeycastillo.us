~b Board 27 -- Go.Live polls server-side (bypass CORS)

Operator reported the live op page was blank. Inspect (inspect-live-page-blank.html) traced it to CORS preflight rejected at Cloud Run IAM. Fix: move the polling out of the browser, into the PowerShell launcher, and have the browser auto-refresh via <meta>.

Shipped · Verified

Root cause

Browser fetch from file:// to https://cadence-...run.app with Authorization: Bearer header triggers a CORS OPTIONS preflight. The preflight carries no Authorization header (by W3C spec). Cloud Run's IAM layer rejects the unauthenticated OPTIONS before the app sees it. Preflight fails → main fetch never sent → page blank.

Fix

Architecture flip. PowerShell loops, fetches /operations/{opId} + /escalate with the gcloud token, renders HTML to $env:TEMP\cadence-live.html, and writes <meta http-equiv="refresh" content="3"> while the op is running (omitted on terminal). The browser just reloads the file from disk every 3 seconds. No browser fetches. No CORS. No token in JS.
PowerShell (Go.Live.ps1) Browser (cadence-live.html on disk) ----------------------------- ------------------------------------ fetch /operations/{opId} (token in PS) fetch /escalate render & write HTML <meta refresh content="3"> sleep 3s; loop reads new HTML from disk ... op succeeded? write without <meta> final state stays put

Verified

Rendered the existing succeeded op reclone-20260601T223202 (Board 26 verification) and confirmed: opId rendered, user=JOEY_BE on the snowflake-auth card, executionStatus=SUCCESS + start/end timestamps + elapsed/bytes/rows, no <meta refresh> tag on terminal op. File size 6.5KB. Browser opened cleanly.
