Bug found + fixed during build

%~dp0 + shift interaction. First Go.Sweep run resolved Go.Driver.bat against C:\metal (the caller's cwd) instead of C:\cadence. Cause: the token parse loop uses shift, which in batch also shifts %0 -- so %~dp0 stopped pointing at the script. Fix: capture set "HERE=%~dp0" at the top before any shift, then call "%HERE%Go.Driver.bat". Same fix baked into all three launchers from the start.
