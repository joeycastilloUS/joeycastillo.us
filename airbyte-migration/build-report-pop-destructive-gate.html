~b Driver HTML report-pop + destructive gate

Two operator features, ~go. Every Driver flow now pops a Be-style HTML report at the end. Destructive steps (DROP, wipe) are flagged and gated -- type 'yes' interactively, or pre-ack with 'iknow' for unattended runs.

Shipped

Feature 1 -- HTML report pops at flow end

New Cadence.Driver/Results/ReportWriter.cs renders the RunResult to a self-contained Be-style HTML report (dark + tron blue, no external deps). Program.cs writes it to results/reports/report-{ts}.html and opens it via Process.Start(UseShellExecute=true) after every flow.

Report sections: summary stats (steps passed, match/total, %, differ, wall), destructive-actions-executed banner (if any), step trail (per-step status/timing/detail), per-stream verdicts (DIFFER list + ERROR list + full collapsible 208-stream table).

Token / flagEffect
(default)report renders + opens in default browser
quiet / --no-reportskip the report entirely
Verified live. Go.Sweep go dry -> wrote report-20260528T040845.html (29.6 KB), opened in browser. Sections present: step trail, per-stream verdicts, DIFFER list. Sweep was 183/208 (88%), 25 DIFFER (all ETHER high-write streams: BARISTA_LOGINS, CART_PRODUCTS, CARTS, ...). Go.Sweep go dry quiet wrote no report (token works).

Feature 2 -- destructive-step gate

New IStep.IsDestructive flag (default false). DropCloneStep + FirestoreWipeStep set it true. The SteeringWheel:

  • Prints !! DESTRUCTIVE STEP -- this deletes data !! in the step header.
  • Interactive: after [Enter]=Go, requires typing yes (not just Enter). Anything else halts the flow before the destructive SQL runs.
  • Unattended (auto / auto-tail / walk-after-gate): HALTS unless the operator pre-acknowledged with the iknow token. With it: proceeds, logs (destructive pre-acknowledged via iknow).
The point: an unattended Go.Rebaseline go can no longer silently DROP the clone. It halts at the Drop step with "needs acknowledgement: re-run with the 'iknow' token." To actually run hands-off through a destructive step you must say so up front: Go.Rebaseline go iknow.
Destructive gate live-test deferred. Reaching DropClone in any flow means the Pause step runs first; Joey's cadence is actively rebuilding its post-reclone trajectory and a test pause would inject a gap. The gate is code-reviewed: auto-mode + no iknow -> ConfirmDestructive returns false -> halt BEFORE the DROP. Interactive -> type-'yes' gate. It will exercise on Joey's next real Reclone/Rebaseline.
