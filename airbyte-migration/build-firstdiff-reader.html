~b /firstdiff endpoint + Go.FirstDiff narrative reader

Reads the per-stream drift timeline and tells the STORY of when each stream first broke from prod -- with schema context, not a table of codes. Caught two streams diverging live on its first run.

Shipped · Deployed

Two parts

PartWhat
GET /firstdiff (service)Walks all snapshots ordered by generatedAt, over the per-tick drifts arrays. Returns per stream: firstSeenAt, firstVerdict, firstSnapshot, stillDrifting (oldest-first) + window + counts. Server-side because the service has FirestoreDb; the launcher just renders.
Go.FirstDiff (launcher)Calls /firstdiff, prints a narrative + pops an HTML report. quiet token = console only.

"More context" -- speaks in operator language

Per the request, the output is a story, not codes. It opens with what was walked, states how many streams diverged and how many still differ, lists each with minutes-since-clone, then a "WHAT THIS MEANS" rollup that annotates each schema in plain English (ETHER = the live POS/ordering app, highest write volume; etc.) and names the lead cause.

CADENCE -- first point of diff Walked 45 snapshots (8 with per-stream drift data), 03:25 -> 07:00 UTC (3.6h). THE STORY: 27 streams have diverged from prod. 27 still differing now. When each one first broke (oldest first): +182m 06:27:45 ETHER.ORDERS Differ (still differing) ... +192m 06:37:03 ETHER.ORDERS_PROMOTIONS Differ (still differing) +192m 06:37:03 ETHER.REFUNDS Differ (still differing) WHAT THIS MEANS: ETHER 27 streams -- the live POS/ordering app. Highest write volume. Lead cause: ETHER. A frozen clone falls behind high-write schemas first. Expected drift, not a bug.
Caught a real first-diff on run one. 25 ETHER streams show 06:27:45 -- that's when per-stream capture went live; they were already drifting. But ETHER.ORDERS_PROMOTIONS and ETHER.REFUNDS show 06:37:03 -- +10 min later. Those two genuinely flipped MATCH->DIFFER after capture started. The reader recorded their true first point of diff. Going forward, every flip is captured precisely; after the next re-clone the whole picture builds from a clean 208/208.
