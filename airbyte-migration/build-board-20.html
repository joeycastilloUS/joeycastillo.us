2. Disposition store + vocabulary (3-state)

Evolved cadence-acks to {disposition, comment, by, ts} (reason kept as a legacy alias; old acks read as explained). The classifier maps each disposition on a would-be Signal:

DispositionEffectVerified live
explained→ Benign "explained: ..."tick → signal drops
accepted-risk→ Benign "accepted risk: ..."tick → signal=0, gate GREEN
blockerstays Signal, reason "BLOCKER: ..."tick → stays red, gate "1 blocker"

A comment is mandatory — every disposition is justified on the record. (All three paths were exercised live, then cleared, so SPREE_CMS_SECTIONS is left un-dispositioned for the operator to decide.)

3. Go.Risk -- the register

Go.Risk list the register (grouped by disposition) Go.Risk <stream> accept "<comment>" real diff, accepted for cutover (-> Benign) Go.Risk <stream> block "<comment>" real diff, must fix first (stays RED) Go.Risk <stream> explain "<comment>" not a real diff -- noise (-> Benign) Go.Risk <stream> clear remove the disposition

Rejects an empty comment. Supersedes Go.Ack (which maps to explain).

4. Honest escalation wiring

POST /escalate/run?stream=SCHEMA.TABLE resolves the stream's business columns (cadence already caches them) and calls the verified /parity/l3 (+ /parity/l1), returning which columns drifted and the row delta. autoEscalate stays opt-in — the 5-min tick never fires it, because each call is a live multi-column Snowflake scan on both DBs.

Go.Escalate cutover gate + signals needing triage Go.Escalate <stream> deep-diff one signal: which columns drifted (+ row delta) L3 verdict: Diff (12 columns profiled, parity /parity/l3 (verified contract)) 12 column(s) drifted: ID, FIT, NAME, TYPE, CONTENT, POSITION, SETTINGS, ... rows: prod=392 clone=393 delta=1 (Diff)

5. Surface + cutover gate

The disposition + gate are surfaced in three places:

  • GET /escalate — cutoverGate, blockers, undispositioned, green, plus each signal's disposition.
  • Go.Signal — prints the gate line + tags each signal [BLOCKER] or [needs triage] + the Go.Escalate / Go.Risk next steps.
  • Public dashboard hero — "ALL CLEAR" vs "N SIGNAL" already IS the gate verdict (green ⇔ signal=0).
The gate. Green = 0 un-dispositioned AND 0 blockers. You can only reach it by digging into each red and either explaining it, accepting the risk with a comment, or fixing it. A blocker stays red on purpose. That is a cutover you can defend.
