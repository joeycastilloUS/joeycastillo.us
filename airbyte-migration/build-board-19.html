1. GET /replay -- one endpoint, three modes

Read-only projection of cadence-snapshots ordered by time, with optional ?from/&to window. The mode is an argument, not a new service:

QueryModeReturns
/replayaggregateper-tick {clean,cosmetic,benign,signal} series + firstSignalAt + baselineCuts
/replay?stream=SstreamS's per-tick verdict + reason + lag; Clean inferred by absence; flips list
/replay?at=Tframenearest snapshot's counts + full drift list (T = ISO time or snapshotId substring)

Pre-driftSchemaVersion=2 ticks degrade honestly: aggregate works on counts (always present), the stream lane reads Unknown before the classifier shipped and labels the v2 boundary rather than guessing.

2. Go.Replay -- the projector

PowerShell launcher modeled on Go.Dash. Pulls /replay, renders inline SVG (no framework), pops the browser, and prints a terminal one-liner.

Go.Replay the cascade movie -- stacked-area verdict mix over time Go.Replay <stream> one stream's verdict lane + flips (when / from->to / why) Go.Replay at <time|snapId> freeze-frame -- counts + drift list at one past tick ... noopen render without opening the browser REPLAY aggregate -- 136 ticks, 05-28 03:25Z -> 21:55Z; first signal 08:34Z; 0 baseline cut(s).

The aggregate render writes the public replay.html; stream/frame are ad-hoc and render to a temp file so exploration never clobbers the published surface.

3. Baseline cuts

A re-clone makes everything match, so the clean fraction jumps sharply. The aggregate walk flags that jump (≥0.30 to ≥0.90 clean) as a baselineCut + draws a "re-clone" marker, so the film doesn't interpolate a lie across a baseline reset. On the current frozen-clone reel: 0 cuts (correct -- no re-clone in the window; the classifier-start jump from 0 to 0.86 is below the 0.90 gate and does not false-trigger). Explicit baselineEpoch on the snapshot doc is the durable follow-up.
