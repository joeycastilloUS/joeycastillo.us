~b Board 23 -- /operations timestamp serialization fix

The Board 22 retro flagged cloneCreatedAt = {} in Go.Watch. Root cause was server-side: Firestore Timestamp objects bypass System.Text.Json defaults and serialize as empty objects. Fixed with a tiny recursive normalizer; every client now sees proper ISO 8601 timestamps.

Shipped · Deployed · Verified

The fix (5 lines)

private static object? NormalizeFs(object? v) => v switch { Timestamp ts => ts.ToDateTime(), IDictionary<string, object> d => d.ToDictionary(kv => kv.Key, kv => NormalizeFs(kv.Value)), IList<object> l => l.Select(NormalizeFs).ToList(), _ => v };

Applied at the two read sites: GET /operations and GET /operations/{opId}. Both now project Firestore docs through the normalizer before returning. Walks the dict/list tree once; Timestamp becomes DateTime, everything else passes through.
