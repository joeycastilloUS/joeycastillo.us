What went well

  • The First Move paid off exactly as designed. Phase 1 was just "return the aggregate series." Once that one array came back ordered and complete (136 ticks), every render mode was downstream presentation. No surprises in phases 2-5.
  • Reusing the /firstdiff walk. The timeline-walk primitive already existed; /replay is the same query with a wider lens. The endpoint compiled clean on the first build (0 warnings, 0 errors).
  • "Clean by absence" was the right model. A stream missing from a tick's drifts array (on a v2 tick) means it was Clean -- no need to store 200 Clean entries per tick. The stream lane reconstructs the full journey from the sparse drift records.
  • Pure SVG held. A 4-series stacked area over 136 points is a handful of <polygon> point strings computed in PowerShell. No framework, no CDN, ages forever -- and it reads.

What was harder / honest edges

  • Pre-classifier ticks are empty. The reel starts at 03:25Z but classification only began ~08:11Z, so the left of the film is blank (total=0) until the v2 boundary. Truthful, but a viewer sees "data starts here." The stream lane labels this as Unknown rather than guessing -- the right call, but it means the movie's first hour is genuinely uninformative.
  • Baseline cuts are unprovable on this reel. The heuristic (clean fraction jumps ≥0.30 to ≥0.90) is correct by construction but the frozen-clone window has no re-clone, so it fired 0 times. It's verified-by-reasoning, not verified-by-observation. The first real re-clone is the true test.
  • The classifier-start jump nearly looked like a cut. Clean went 0 → 0.86 when classification began -- a 0.86 jump. Only the ≥0.90 ceiling kept it from false-flagging as a re-clone. That gate is load-bearing and a touch lucky; the explicit baselineEpoch field would remove the guesswork.

What to do differently / next

  • Add baselineEpoch to the snapshot doc (the planned follow-up). Replace the heuristic cut detection with an explicit segment boundary so re-clones never rely on a fraction-jump gate.
  • Consider clamping the aggregate window to the v2 boundary by default so the movie opens on informative data, with a flag to show the full raw reel.
  • The recovery reel is the real payoff and is still unfilmed. Everything so far is the cascade (clone falling behind). When airbyte-philz feeds the clone, replay should show signals collapsing back to clean -- that footage IS the cutover proof.
Where cadence stands. Boards 16-19 built the full observability surface: classify every diff (16), make it fast + ackable + escalatable (17), surface only genuine signal (18), and now replay the whole timeline (19).
