Build Go.Replay

Execute be/designs/2026-05-28-2135-design-go-replay.html. A time machine over the verdict timeline: one server endpoint projects the recorded snapshot series, one launcher renders it three ways — aggregate movie, per-stream lane, freeze-frame. Pure read over data /tick already writes.

Plan — Awaiting Approval
Intent: Go.Signal says what's wrong now; Go.FirstDiff says when each broke; Go.Replay shows the whole story unfold — the clone falling out of sync stream by stream, and (once fed) the signals collapsing back to clean. The cutover, on film.

Phases

1 GET /replay aggregate series (First Move)

Server reads cadence-snapshots ordered by time, projects [{snapshotId, time, clean, cosmetic, benign, signal}, ...]. Optional ?from= / ?to= window. Mirrors the /firstdiff walk. No render yet.
Accept: GET /replay returns the full 127-point curve, time-ordered, counts summing to total per tick. The movie's data exists in one call.

2 Per-stream projection (?stream=)

For a given stream, walk the same series and emit [{time, verdict, reason, lagHours}, ...]. Clean inferred by absence from a tick's drifts array. Pre-driftSchemaVersion=2 ticks degrade to verdict-only; clamp + label the boundary.
Accept: /replay?stream=ECOMM.SPREE_CMS_SECTIONS returns its journey -- Clean ticks, then the Benign-stale stretch, then Signal -- each with reason where v2 data exists.

3 Go.Replay aggregate renderer

PowerShell launcher (mirrors Go.Dash): pull /replay, build an inline-SVG stacked-area / multi-line of clean/cosmetic/benign/signal over time, write HTML to the site path, pop it. Print a terminal one-liner: window span, tick count, first-signal time.
Accept: Go.Replay opens an HTML time-lapse showing benign growing + the signal spike as the frozen clone ages; terminal prints the one-line summary. No framework, pure SVG.

4 Stream lane + freeze-frame

Go.Replay <stream>: a horizontal verdict-colored lane across the window with flip markers annotated when + why. Go.Replay at <time>: reconstruct one tick's full picture (counts + drift list). Both reuse the Phase 1/2 endpoint params.
Accept: The lane shows SPREE_CMS_SECTIONS flip Clean→Benign→Signal at the right ticks with reasons; at 16:53Z reproduces that tick's verdict counts.

5 Baseline cuts + publish

Detect a re-clone heuristically (a tick where clean jumps to ~total) and draw a cut marker so the film doesn't interpolate across a baseline reset. Publish a representative replay to joeycastillo.us/airbyte-migration/; render build report; update index + telemetry; close Board 19.
Accept: Replay marks any baseline cut in the window; a published replay is live; board closed, DoD green.
