Build human-friendly Driver CLI — verb launchers

Execute be/designs/2026-05-27-1905-design-driver-cli.html. Per-verb .bat launchers translate positional mode tokens to existing Go.Driver flags. ~11 pts, one board, no cadence-driver.exe parser change (only a help pointer).

Plan — Awaiting Approval
Intent: Replace Go.Driver --flow X --skip-write --unattended -i with Go.Sweep, Go.Reclone, Go.Rebaseline + positional tokens (ask|go|walk|manual, dry|live). Each launcher is a thin .bat shell that maps words to flags and calls Go.Driver. One source of truth (the .exe), multiple ergonomic faces.

Scope

In scopeOut of scope
Go.Sweep.bat, Go.Reclone.bat, Go.Rebaseline.bat in C:\cadence. Positional token parser (ask/go/walk/manual + dry/live), unknown-token rejection with expected-list, per-launcher help. Go.Driver --help pointer to the new launchers. All shell out to the existing Go.Driver.bat -> cadence-driver.exe. Rewriting the cadence-driver.exe flag parser (kept; launchers translate to it). PATH installation. Changing defaults (writes live, prompts default). Go.Pause/Go.Resume/Go.Wipe launchers (deferred). Touching Go.Status/Go.Diff (already right shape).

Approach

  1. Go.Sweep first (First Move). Build the token parser skeleton once. Prove the translation end-to-end with a real read-only run (go dry).
  2. Clone to Reclone + Rebaseline. Same skeleton, different FLOW/VERB/DESC. Help + validation smoke (real runs are destructive).
  3. Discoverability. Point Go.Driver --help at the verb launchers so the flag form is clearly the escape hatch.

Phases

1 Go.Sweep.bat -- token parser skeleton (First Move)

Parse ask/go/walk/manual + dry/live -> --interactive/--auto/(--interactive --unattended)/--manual + --skip-write. Capture %~dp0 before parse. Unknown-token guard. Help text.
Accept: Go.Sweep help prints usage. Go.Sweep zonk exits 2 with expected-list. Go.Sweep go dry runs a real read-only sweep end-to-end (translation = --auto --skip-write).

2 Go.Reclone.bat + Go.Rebaseline.bat

Clone the Sweep skeleton. FLOW=reclone / reset-baseline. Tailored help (dry inert on reclone, discouraged on rebaseline).
Accept: help + bad-token verified on both. Skeleton identical to the proven Sweep template; real runs deferred (destructive).

3 Go.Driver --help pointer

Add a "Friendlier verb launchers (preferred)" block to the exe --help listing Go.Sweep/Reclone/Rebaseline/Status/Diff. Flag form labelled the low-level escape hatch.
Accept: Go.Driver --help shows the launcher block. Release exe rebuilt.

Constraints

  • No new dependencies -- pure .bat + the existing .exe. Pure Metal Test from the design: 1 load-bearing dep (cadence-driver.exe).
  • %~dp0 must be captured before any shift (shift also shifts %0).
  • Launchers live in C:\cadence alongside the Go.X family. No PATH change.
  • Defaults unchanged: interactive prompts, live Firestore writes.
