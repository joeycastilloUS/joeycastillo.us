Pattern: Each attempt takes 255–305 min regardless of date. The per-attempt speed hasn't changed. But from April 8 onward, attempts 0 and 1 crash, forcing 3 attempts per job. 3 × 270 = 810 min (13.5 hours) — that's what it looks like when you see the jobs "slowing down."

Root Cause: custom_wholesale Crashes

Every failed attempt dies on the same stream with the same error:

custom_wholesale: AirbyteTracedException("'NoneType' object is not iterable")

The source-netsuite:0.4.1 connector hits a null value in the NetSuite SuiteQL API response that it doesn't handle. This crashes the source, which crashes the entire sync attempt.

The error is intermittent — attempt 2 (the final retry) usually succeeds. This suggests a specific record or page that sometimes returns null depending on NetSuite's internal state, query timing, or concurrent access.

NetSuite returns a 500 Internal Server Error on the SuiteQL endpoint, the connector backs off and retries, but then gets a 401 Unauthorized — the OAuth token likely expired during the 2+ hour wholesale read. The connector doesn't re-authenticate mid-stream.

Action Items

1. Fix custom_wholesale NoneType crash. The source-netsuite:0.4.1 connector doesn't handle null values in wholesale SuiteQL responses. Options: (a) patch the connector to handle None gracefully, (b) identify the specific wholesale record with null data and fix it in NetSuite, (c) add null-safe parsing to the SuiteQL query.
2. Switch full_refresh streams to incremental. custom_sales_order and custom_production already have lastmodifieddate cursor fields configured but are set to full_refresh. Switch to incremental/append_dedup. This alone would cut sync time from 270 min to ~130 min.
3. Investigate why custom_wholesale is 75x slower. 44K records at 300 rec/min suggests the underlying SuiteQL query is doing expensive joins or scanning a massive table. Check the actual SuiteQL query in the connector config.
4. Investigate OAuth token expiry. The 500 → 401 pattern suggests the OAuth token expires while wholesale grinds through its 2+ hour read. The connector should re-authenticate, but it doesn't.
5. Consider separating custom_wholesale. If wholesale stays slow and intermittently fails, isolating it into its own connection means the other 5 streams sync independently.
