Design points realized

  • Cursor excluded from hash, kept in filter. A row's cursor is the field most subject to UPSERT churn; hashing it would surface every cursor-only-change as DIFFER. We filter ON cursor to scope to settled rows, but don't include it in the hash.
  • Cast fallback for non-timestamp cursors. When a cursor is bigint ID or similar, TO_TIMESTAMP cast fails. The helper catches the specific error and falls back to whole-table comparison, marked filter_mode='fallback-non-ts' in the result.
  • Magnitude on demand. EXCEPT COUNT runs only when include_magnitude=True AND verdict is DIFFER. Keeps the default snapshot fast; opt in for forensic detail.
  • Graceful errors. Unrecoverable failures (missing column, permission, network) return verdict='ERROR' with the error string in error. Caller decides whether to retry, skip, or surface.
