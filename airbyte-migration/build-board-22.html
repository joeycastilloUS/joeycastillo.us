1. cadence-operations log + read endpoints

GET /operations?limit=N (recent ops) and GET /operations/{opId} (one op's full trail). Doc shape: {opId, kind, status, startedAt, finishedAt, steps[], cloneCreatedAt?, error?}. Steps carry {name, status, startedAt?, finishedAt?, error?} so a watcher knows the exact step + duration.

2. POST /reset/reclone -- async harness

X-Reset-Confirm guarded, ResetLock single-flight. Spawns a background Task.Run that walks the four steps, writing each transition to Firestore. Returns 202 Accepted + {opId} immediately; the request doesn't block on the 15-minute reclone. Cloud Run instance survives the work (no recycle observed); if it ever recycles mid-op, the doc records the last good step for manual resume.

3. Drop AIRBYTE_CLONE step

DROP DATABASE IF EXISTS AIRBYTE_CLONE via the existing SnowflakeConn.OpenAsync + 3600s CommandTimeout. SQL lifted verbatim from Cadence.Driver/Steps/DropCloneStep.cs. On failure, step.error captured + op halted before Create. Live: 2m 3s.

4. Create AIRBYTE_CLONE CLONE AIRBYTE_DB step
