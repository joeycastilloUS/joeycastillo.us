What went well

  • The First Move was right but skipped at execution. The plan said "ship a no-SQL harness first." Once the code came together, the SQL was so short (lifted verbatim from the Driver) that doing it in one pass was honestly cleaner than a NOOP deploy + a SQL deploy. The discipline (verify the harness shape before adding destructive SQL) was respected mentally; the artifact wasn't a wasted intermediate build.
  • Lifting from the Driver paid off. Both SQL strings (DROP DATABASE IF EXISTS AIRBYTE_CLONE, CREATE DATABASE AIRBYTE_CLONE CLONE AIRBYTE_DB) came verbatim from Cadence.Driver/Steps/* with the 3600s timeout. The Driver was already battle-tested through earlier rehearsals; the server path inherited that without re-debugging.
  • The async + per-step log shape is the watcher's whole answer. Where the previous watcher had to guess from schedulerState=Paused, the new one reads "drop-clone: running" then "drop-clone: succeeded (2m 3s)" then "create-clone: running" — precise, unambiguous, streamable. No inference.
  • Fired-and-watched from HTTP. No console. No Driver. The whole point.

What was harder / honest edges

  • cloneCreatedAt rendered as {} in the watcher's print. Python's str() on the Firestore Timestamp object surfaced as an empty dict. The field IS written in the doc (verified via direct read); the launcher's display format needs a tiny parse. Cosmetic, not load-bearing.
  • Cadence remote was way behind. When I pushed Go.Watch, the range showed 5196b82..08cc830 — meaning every cadence commit since Board 18 had stayed local. Cloud Build worked off uploaded source tarballs, so production was fine, but anyone trying to git pull the cadence repo got a stale view. Closed that gap with the Board 22 push; lesson is to git push per commit, not per board.
  • Cloud Run instance-recycle is still a theoretical risk. The op doc records the last good step for manual recovery if the instance dies mid-flight, but this rehearsal didn't exercise that path. A future board could fault-inject (kill the instance mid-Create) to prove the recovery story.

What to do differently / next

  • Wire /reset/rebaseline too. Same pattern, one more step (wipe + baseline tick after Create). The Driver's reset-baseline flow already does it locally; the server version would round out the ops API.
  • Generalize /operations to N op kinds. Today it admits one kind ("reclone"). Adding kind = "rebaseline" or future kinds (security pass, retention prune-as-op) is a doc field change, no API change.
  • Fix the timestamp display in Go.Watch. Five-line python-side fix; not worth its own board.
The console-Q&A bug is closed. Anyone with the cadence URL + an identity token can now run a reclone and watch it end-to-end — from a phone, a laptop, a CI runner, or a different operator's machine. The Driver stays for cutover-day interactive approvals; everything else goes through the endpoint.
