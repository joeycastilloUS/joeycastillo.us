§2 — a-d-d verdict

NO mention of an "identity skill" anywhere. The 6 specced security skills are crypt, auth, seal, vault, policy, wipe. Identity responsibilities are correctly split:

  • crypt — Ed25519 primitives (sign/verify/keypair), X25519, AES-GCM, SHA-256, PBKDF2 — already implemented in nous
  • auth — TOTP, sessions, nonce ring, API keys, Cloudflare domain verification — already implemented
  • relay/CLAUDE.md — Node Identity model: operator's Ed25519 pubkey compiled into binary, each node carries node.key + node.pub + node.cert (operator signature over pubkey+role) — this IS the identity model

§4 — Open-source verdict

Canonical Ed25519 + X25519 pure-C libraries (orlp/ed25519, compact25519, Micro25519) — all are point utilities, not "frameworks." NIST/Venafi trust-anchor guidance: compile the trust root into the binary. That's exactly what nous does. No open-source pattern suggests a separate "identity framework" is needed.
