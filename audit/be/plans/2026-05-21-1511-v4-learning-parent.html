Every architectural and conceptual claim in api's purpose/learning/CLAUDE.md is either covered or genuinely superseded by ks's nous/intelligence/ docs. ks deliberately decomposed api's monolithic "learning" into intelligence/memory/security: store moved to memory, crypto to security, web folded into acquire. What api purpose/learning/CLAUDE.md contains (51 lines)

  • Purpose statement: "Autonomous knowledge acquisition. Ask questions, search the web, digest facts, reason over what you know. Learning is the brain that understands language and grows its knowledge."
  • What it is: "A knowledge engine backed by a triple store. Natural language in, structured knowledge out. When local knowledge is insufficient, it autonomously learns from Wikipedia and DuckDuckGo."
  • Architecture tree: 5 skills — store, understand, reason, web, crypto; builds liblearning.a + test binary.
  • Data model: "Same binary format as memory's store — designed for shared-memory at the intent level." FileHeader 24B with magic 0x50545300; Fact 24B (three uint64); DictEntry 16B; FNV-1a constants offset 0xcbf29ce484222325, prime 0x100000001b3.
  • Storage files: facts.bin, dict.bin, strings.bin with sizes per record.
  • Pipeline: Question → understand() → reason() → [web learning if confidence < 0.6] → Answer — note the explicit 0.6 confidence threshold.
  • Repo pointer: ai-pantheon-internal/purpose-learning (frozen).

What ks's intelligence stack contains

  • nous/CLAUDE.md — top-level NOUS intent doc. Vision/mission/nodes. Intelligence listed as: "The brains. 28 operators = complete instruction set. eval() = understand() + reason(). Four execution levels."
  • nous/intelligence/CLAUDE.md — reframes "learning" as "intelligence" with the universal detect → decode → apply pattern. 6 skills: language, understand, learn, acquire, reason, read (no store, no web, no crypto — moved out). Covers BASICI, eval(), 28 operators, escalation loop, zones, /search algorithm.
  • EVERYTHING_IS_TRIPLE.md — 10 triple categories, four stores (Memory/Intelligence/Directory/Search), the query engine model.
