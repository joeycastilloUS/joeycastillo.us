~i AP-17 — pantheon-gateway: Pantheon-era external API, ARCHIVE.

Investigated 2026-05-25 22:40 PDT. ai-pantheon/pantheon-gateway is a Cloudflare Worker (JS, 1,273 LOC, v2.2.0) that fronts the dormant "Pantheon Brain" knowledge base. It is not clauderun's API gateway, not a nous component, and both it and its backend are offline. ARCHIVE
The queue question: "is this clauderun's API gateway?" — No. pantheon-gateway is the external agent API for the Pantheon knowledge-base brain (a 680M-fact triple store + Ollama LLM on a GCE VM). That is a different layer from clauderun, which is the agent-execution/dispatch platform (AP-12/13/14/15/16). They share the "Pantheon" name and nothing else. Disposition is therefore ARCHIVE, not move-to-clauderun.

§1 — What pantheon-gateway actually is

A single Cloudflare Worker (src/worker.js, 1,273 LOC) bound to a KV namespace (GATEWAY_KV) and a brain backend (PANTHEON_BRAIN_URL = http://35.197.77.81.nip.io, a GCE VM). Created 2026-02-19, last pushed 2026-02-22 — dormant ~3 months. Heavy Pantheon Inc. / ai-pantheon.ai branding throughout.

§2 — a-d-d verdict (nous fit)

Same architectural mismatch that decided AP-10, AP-11, AP-13: nous is pure-C bare-metal; this is JavaScript on Cloudflare Workers. The one conceptual overlap — a "triple store" — is superficial. nous's triple store (nous/memory) is on-node, pure-C, single-cycle (S,P,?) lookup. pantheon-gateway's "store" is a remote 680M-fact cloud brain with an LLM bolted on. Wrong language, wrong runtime, wrong locus. Zero merge value into nous.

§2 — Source verdict (clauderun fit + brain dependency)

The plan flagged this as a possible "clauderun API gateway." It is not. clauderun runs and dispatches agents; pantheon-gateway hands external agents (Grok, etc.) metered access to a knowledge base. The backend it depends on — the Pantheon Brain at 35.197.77.81 — is not part of the consolidated nous/clauderun architecture and is itself offline (see §4). Folding a dead JS gateway for a dead cloud brain into clauderun imports liability, not capability.
