§3 — MH-1 replay/rollback guard — REAL & CORRECT

  • mesh_roster_meta (mesh_roster.c:90-110) — extracts signed_at/version via strtoull, first-match-wins.
  • mesh_roster_freshness_check (:129-141) — reads <path>.seen sidecar, rejects signed_at < last (:136), advances on accept (:139). Comparison is >= accept / strict < reject — right call: idempotent reboot of the current roster allowed, anything strictly older blocked. Missing sidecar → last=0 → first roster accepted.
  • Order of operations correct (mesh_roster_load :147-171): signature verify FIRST :160, then freshness :167, then parse :170. A forged old roster never reaches the freshness gate.
  • Tests real: 20k-iteration fuzz test_mesh_e2e.c:101 (no crash, zero forged accepts :112-113) + full e2e chain with the load-bearing "older REJECTED" assert :87; unit rollback matrix test_mesh_roster.c:138-169.

Three minor hardening gaps in MH-1 (low severity, consistent with threat model): (1) read-then-write of .seen is non-atomic / no lock — harmless single-threaded at boot, real if ever concurrent; (2) the sidecar is unauthenticated plaintext — a local attacker with FS write can zero it to defeat anti-rollback (but local FS write = already game over); (3) seen_write failure is silent (:122-124 returns void) — on an unwritable dir the roster is accepted but the high-water mark never advances, degrading rollback protection to a no-op rather than failing closed. Code is ahead of spec here — the v1 relay spec does not yet document the freshness guard; spec should be updated to match (flow-up before down, Rule 11).

§5 — Operator private key handling — WEAK on all three axes

  • Zeroization — elidable. Key IS wiped after use but via plain memset/loops (operator-keygen/main.c:136, mesh-roster-build/main.c:189, operator-sign-node/main.c:69,77,82,100). No explicit_bzero/memset_s/SecureZeroMemory/volatile anywhere in C:\nous. Under C11 + -O2 these are dead stores the optimizer may remove. Stack buffers, no mlock → pageable. The skill that would provide a real secure-zero (wipe) is the unbuilt one.
  • File perms — Unix-only. chmod(path,0600) guarded #ifndef _WIN32 (operator-keygen/main.c:64-68); Windows branch is (void)restrict_mode; — no SetNamedSecurityInfo/icacls/DACL. Only Windows "protection" is an advisory printf at :117. Confirms handoff.
  • At rest — plaintext, never logged. Raw 64-byte key written cleartext to $HOME/.nous/operator.key (:108); no encryption. Never printed/logged (all output is paths, public hex, counts). Windows path is weakest: plaintext root key + no ACL + elidable wipe.
