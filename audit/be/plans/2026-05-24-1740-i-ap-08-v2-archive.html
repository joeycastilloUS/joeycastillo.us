§2 — a-d-d spec verdict

Grep for: Multiboot, Multiboot2, GRUB, Pi VM, bytecode, register machine,
            VM interpreter across kastil-systems/a-d-d

Result: ZERO matches. The a-d-d does not call for Multiboot2 or for a
        bytecode VM. Both are orphan concepts not in the canonical spec.

Smoking gun: pnos.c contains this comment:
  "Core 1+: Reserved for Pi VM workers (future)"

That's a PLACEHOLDER. Future intent, not current requirement.

kastil-systems/target-runtimes already owns the production boot.asm (148 lines, ESP=64MB stack — tuned beyond assembly's 16MB version). Linker script + pnos.c + Makefile.biRT all baked into a single 1MB disk image. Zero references to piasm, .pi.enc, Pi VM bytecode, GRUB, or Multiboot2 anywhere in the kastil-systems active code.
