§2 — a-d-d verdict

nous/security/skill/policy exists as docs only (NOTE.md). Per a-d-d: reserved for object-level primitives — tear-line stripping (auto-hide fields above recipient's clearance), revocation (originator revokes access), per-object access control.

That's NOT what sentinel does. sentinel does runtime behavior monitoring of autonomous agents. Wrong scope.

§3 — Source verdict

nous architecture today (per MESH_STATUS.md): 4 primes + 1 relay, all synchronous request/reply. Zero autonomous agents. No background tasks. No daemons monitoring other daemons. There's nothing for sentinel to oversee — every operation in nous is bounded by a single client request lifecycle.

nous/comms/skill/waf (pure C, already implemented) handles request-edge: rate limiting, path validation, origin allowlist. That's the only "monitoring" surface nous needs.
