§4 — Why merging is wrong (4 reasons)

  1. Language barrier is insurmountable. Python + Solidity will not compile into pure-C. Rewriting from scratch (as we did with AP-09 integrity) discards safety's code anyway — there's no code to "merge."
  2. nous already solves the problems safety solves, better. nous vault has brute-force defense safety lacks. nous crypt has HW-accelerated AES-GCM safety doesn't. The padding/shuffle/tokens concepts are traffic-analysis defenses for client→server architectures — nous operates peer-to-peer with end-to-end encryption already at link layer.
  3. The pieces unique to safety contradict nous's design. Distributed seal authority means trusting 4 of 7 blockchains — that's adding 7 trusted third parties. nous starts from "all external systems are untrusted." Your key lives in RAM, dies with power. There is no recovery; there is only continuation or wipe.
  4. safety addresses a different threat model. safety = "encrypt my data so a server can't read it" + "obfuscate traffic so an observer can't fingerprint it." nous = "encrypt a military-grade mesh so captured nodes reveal nothing." The second is a harder problem with a different answer (volatile keys + crypto-shred, not distributed recovery).
