Patterns Found

DRIFT a-d-d exists in SIX places — your "single source of truth" isn't single

CLAUDE.md (kernel) declares kastil-systems/a-d-d/GLOBAL.md as the methodology source. Reality:

  • kastil-systems/a-d-d — declared canonical (Batchfile, 2026-05-19)
  • acmedevUS/a-d-d — description literally says "single source of truth" (Batchfile, 2026-03-11)
  • acmedevUS/a-d-d-template — archived
  • Life-Water/a-d-d — Batchfile, 2026-03-18
  • ai-pantheon-internal/ADD — Shell, 2026-02-24, capitalized
  • joeycastilloUS/a-d-d-guidance — HTML, 2026-05-10

This is the #1 risk to your "load from metal and work anywhere" goal. Six copies means six diverging methodologies. The one with the most recent push isn't automatically right.

DRIFT metal-console exists in TWO orgs with TWO different primary languages

  • joeycastilloUS/metal-console — HTML, 2026-05-19. The operator console (this repo).
  • kastil-systems/metal-console — C, 2026-05-18. Different codebase entirely.

Both active. Both nameless (no description). One is presumably the runtime (C); the other is the operator surface (HTML). When you say "metal" in conversation, you don't know which you mean — and neither does anyone reading the org list.

CONTAMINATION acmedevUS holds repos that don't belong to Acme

acmedevUS is supposed to be Acme Dev LLC commercial work. It currently also holds:

  • ai-pantheon — Pantheon brand, wrong org
  • claude-run-storyline — Pantheon product, wrong org
  • employee-validator — "Crew Check ... PWA for Philz Coffee" — wrong org
  • crew-check-api — "Crew Check API ... Philz Coffee" — wrong org
  • claudius-maximus — unclear, probably Pantheon
  • a-d-d — kastil-systems methodology, wrong org

Pattern: Acme used to be where everything started. It's still where the residue lives. The split happened, the move didn't.

PARKING LOT ai-pantheon was mass-created and mostly abandoned

28 repos in ai-pantheon. 21 of them have a last-push date of 2026-02-22 — the same day. That's a create-spree, not active development. Names like pantheon-mesh, pantheon-mail, pantheon-personal, pantheon-worker-alpha/beta/gamma — empty placeholders.

The Pantheon thinking is real (it's in the kernel doc), but the GitHub structure froze at sketch stage. Decision needed: fold the live work back to a small set (maybe pantheon-platform, pantheon-web, the public trio) and archive the rest, or accept the scaffold and revive it deliberately.

  • metal-desktop — described, clear ("browser-based cockpit on localhost:5050")
  • ai-metal — no description

Two more confusables: joeycastilloUS/kastil-systems (a repo named after the org) and joeycastilloUS/kASTIL. Both empty descriptions. Reading the names cold, you can't tell which is which.

  • overwatch_app in acmedevUS AND Mendocino-Farms — likely one is a fork or stale duplicate.

How to use this audit

Two ways:

  1. Make the moves yourself — work through the 10 recommended moves above in order. Each one is small. The whole list is a weekend.
  2. Convert to boards — run ~e on this report and the recommended moves become Fibonacci-sized board items in be/board.md. Then ~b pulls them one at a time. That's the "load from metal and work on anything" pattern Joey wanted.

Either way, re-run ~i in this repo after each pass. The audit is the discipline — every move shows up in the next report.
