§4 — Known open security gaps (from the deep audit)

The mesh trust layer is "Nebula-grade foundation, honestly v1." Built: authenticity + integrity + (MH-1) replay/rollback guard. Not built (designed + estimated, MH-2..MH-5):

  • No cert/roster expiry — certs live forever (MH-2: valid_until, 30d TTL).
  • No revocation — can't kill a leaked node/key without recompiling all (MH-3: signed revocation list).
  • No per-peer mutual auth — relay↔prime trust by IP + shared PSK, not identity (MH-4: hand-rolled challenge-response).
  • No key rotation ceremony + roster is signed-plaintext (MH-5: operator-key versioning + sign-then-encrypt).
  • Test system gaps — 70 test files, ~480 asserts, but no unified runner / CI / coverage (see test ~i/~p). keygen + roster-build had 0 standalone unit tests (now covered by e2e interop).
  • Operator key on Windows — chmod 0600 Unix-only; no ACL enforced on Windows.
