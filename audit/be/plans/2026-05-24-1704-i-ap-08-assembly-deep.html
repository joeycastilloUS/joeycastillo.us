§4 — What's net-new (the value)

Of assembly's 1,372 LOC:

  • boot.S (49 LOC) — Multiboot2 entry. Enables boot via GRUB, QEMU's -kernel mode, and other modern bootloaders. target-runtimes is BIOS-MBR-only today; this opens new test/dev scenarios.
  • Pi VM bytecode (858 LOC across 7 programs) — A custom 64-bit register VM with 16 GPRs, 8-byte fixed instruction format, ~40 opcodes (arith/logic/control/crypto/syscall). Programs include arithmetic, conditionals, function calls, iterative Fibonacci, a 559-line transaction gateway (receiver.pi.asm), and bytecode-level mesh routing.
  • The piasm compiler is REFERENCED but NOT in the repo — the .pi.asm files are source; the binary toolchain that compiles them to encrypted .pi.enc bytecode lives somewhere else. Joey would need to find/include that tool for the Pi VM to be runnable, OR write a fresh assembler in pure C if going forward.

Conceptual value: Pi VM is a portable bytecode layer that could run on any bare-metal or hypervisor target — separate from target-runtimes' native C kernel. It's reference/demo material that documents intended behavior of the mesh protocol in a more readable form than C.

§7 — Open question

The piasm compiler that turns .pi.asm source into runnable .pi.enc bytecode is referenced in the README but not present in the repo. Without it, the Pi VM programs are documentation only. Joey decides: hunt down piasm and bring it too? Write a fresh assembler in pure C as a tools/ entry? Or accept the .pi.asm files as reference/spec material only?

§8 — Risks

  • Makefile path references — if boot.S goes into target-runtimes/boot/ (vs root), variants (mesh-gvnic-learning, hex-tecno-km7k-sensing) need their relative paths updated.
  • Symbol collisions — boot.S defines Multiboot2 header symbols; check no clash with boot.asm's existing symbols. (Unlikely — they target different boot mechanisms — but verify at link time.)
  • Pi VM tooling gap — without piasm, the .pi.asm files build nothing. Make the move scope clear (we're moving the .pi.asm sources, not promising to make them runnable).
