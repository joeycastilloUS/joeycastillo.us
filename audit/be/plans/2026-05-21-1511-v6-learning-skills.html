### Triple Store Binary Format

Identical across all four triple stores (Memory, Intelligence, Directory, Search)
so any purpose can mmap any other's facts at the byte level.

- FileHeader (24 bytes): magic `0x50545300` ("PTS\0"), uint32 version,
  uint64 count, uint64 capacity
- `facts.bin` = FileHeader + array of 24-byte Fact (3 × uint64: subject,
  predicate, object)
- `dict.bin` = FileHeader + array of 16-byte DictEntry (uint64 id,
  uint32 offset, uint32 length)
- `strings.bin` = FileHeader + contiguous NUL-terminated string blob
- FNV-1a 64-bit hash: offset `0xcbf29ce484222325`, prime `0x100000001b3`
- Auto-grow files 2× when full
- Initial capacity: 8192 facts
- `store_build_index` for O(1) lookup at 30M+ facts
