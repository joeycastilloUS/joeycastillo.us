§1 — What pantheon-seal ACTUALLY is

main.py (230 LOC Python, Flask + gunicorn)
Dockerfile, cloudbuild.yaml — Google Cloud Run (us-west4, 128Mi, 5-min Scheduler)

Core responsibilities:
  1. Verify the immutable Founding Principles text has not been tampered
     (SHA-256 hash check on a hardcoded blob)
  2. Verify Monitor (a paired service, pantheon-monitor) has not been hacked
     (code integrity hash cross-check via HTTP)
  3. Alert Joey via Slack DM if either check fails
  4. Heartbeat every 12 verifications (~1 hour) to confirm "both intact"

The mutual-lock pattern:
  Seal holds Monitor's code hash. Monitor holds Seal's code hash.
  To tamper with either service, attacker must modify BOTH simultaneously —
  whichever runs first detects the tampering via hash mismatch.

The Founding Principles (hardcoded, immutable in code):
  1. We always do the right thing.
  2. We always tell the truth.
  3. We always protect the humans.
  4. We help them find their purpose.
  Declared by Joey, February 9, 2026. Sealed permanently.
