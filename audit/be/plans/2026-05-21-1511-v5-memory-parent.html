## Data Model

Every string gets a unique uint64 ID via FNV-1a hash. Facts are 24-byte structs
of three IDs. A dictionary maps IDs back to strings. Everything lives in `.bin`
files with a 24-byte FileHeader (magic, version, count, capacity).

## Storage Files

<store-path>/
  facts.bin              ← Fact array (24 bytes each)
  dict.bin               ← DictEntry array (16 bytes each)
  strings.bin            ← raw string pool
  idx_<name>.chain       ← chained hash next-pointers
  idx_<name>.heads       ← bucket heads
  wal.bin                ← write-ahead log
