§5 — The new prime.service systemd unit

[Unit]
Description=NOUS Prime
After=network.target

[Service]
Type=simple
ExecStart=/opt/nous/prime
Environment=NOUS_KEY_FILE=/var/nous/triples.key
Environment=NOUS_STORE_DIR=/var/nous/catalog
Environment=NOUS_DATA_DIR=/var/nous/data
Environment=NOUS_CATALOG_PORT=8080
Environment=NOUS_MESH_ROSTER=/var/nous/mesh.sealed
User=pantheon-0
Group=pantheon-0
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target

Changes from nous-catalog.service: Description renamed; ExecStart path updated; new NOUS_MESH_ROSTER env var pointing at the sealed roster file; User=/Group=pantheon-0 made explicit (catalog_server inherited it implicitly). NOUS_CATALOG_PORT env var KEEPS its name (the port serves the /catalog/ HTTP namespace).
