§2 — Trust chain verified

  Operator (Joey's machine, ~/.nous/)
    ────────────────────────────────────────────
    operator.key  64 bytes priv  ← never leaves this machine
    operator.pub  32 bytes pub
                  ↓
                  embedded in security/operator_pub.h
                  ↓
                  compiled into every prime binary
                  ↓
  Operator signs mesh.sealed via mesh-roster-build
    ────────────────────────────────────────────
    payload: 25 triples (5 nodes × 5 triples each:
             type/role/address/signed_at/version)
    magic:   "MESHROSTER1\0\0\0\0\0"
    sig:     ed25519_sign(operator_priv, magic||len||payload) = 64 bytes
    total:   878 bytes
                  ↓
  Distributed to /var/nous/mesh.sealed on each prime VM
                  ↓
  Prime boots: mesh_roster_load(store, "/var/nous/mesh.sealed")
    ────────────────────────────────────────────
    1. read 878 bytes
    2. verify magic "MESHROSTER1"
    3. ed25519_verify(OPERATOR_PUB, magic||len||payload, sig) → OK
    4. parse null-delimited triples
    5. triple_store_insert() × 25
    6. log: "Mesh roster: loaded 25 node triples from mesh.sealed"
                  ↓
  Prime now knows: every peer's name, role, address, version
  All 4 primes know the same mesh, signed by the same operator,
  verified independently on each VM. Indestructible substrate.

To rollback on any VM:
  sudo systemctl stop prime
  sudo systemctl disable prime
  sudo mv /opt/nous/prime /opt/nous/prime.broken
  sudo cp /opt/nous/<layout-specific path>.bak  /opt/nous/<orig path>
  sudo cp /etc/systemd/system/nous-catalog.service.bak  /etc/systemd/system/nous-catalog.service
  sudo systemctl daemon-reload
  sudo systemctl enable --now nous-catalog
  sudo systemctl status nous-catalog
