Build script bugs discovered (TODO commit fixes back to nous)

  • intelligence/build_linux.sh has a hardcoded cd /mnt/c/kastil/nous/nous on line 4 (author's WSL path) — breaks any build outside that path. Patched out for this build; should be removed in the committed source.
  • intelligence/build_linux.sh doesn't include -Isecurity on the final prime_server.c compile line — newly-required since mesh_roster_format.h moved to security/ on 2026-05-30. Patched out for this build.
  • The script also references several .o files (language.o, understand.o, learn.o, read.o, acquire.o, hash.o, mmap.o, index.o, journal.o, scope.o, store.o, pack.o) it doesn't build — so it isn't the canonical builder anyway.   • All shell scripts have CRLF line endings (Windows artifact) — Linux bash chokes. Need a .gitattributes entry forcing LF for *.sh.

What's enforced now that wasn't 5 minutes ago

  • An attacker who somehow obtained a validly-signed OLDER roster can no longer downgrade the mesh — MH-1's .seen sidecar will reject anything with an earlier signed_at than the last accepted roster.
  • If we forget to re-roll the roster before 2026-06-29, primes will refuse to load the expired roster and graceful-degrade (continue with the previously-loaded triples, but reject the now-stale file). The 30-day TTL is now real enforcement, not advisory metadata.
  • If a node is compromised and added to the next roster as revoked, every prime will purge that node's triples from its store at load time. The CRL primitive is live.
