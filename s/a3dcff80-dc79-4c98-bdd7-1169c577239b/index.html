Does the new one fit? Not as "ticket."

Read the list top to bottom: understand · learn · read · reason · reconsider · summarize · judge · acquire. Every intelligence skill is a capability, and every capability is data-agnostic — understand doesn't care if it's a ticket or a contract; learn writes any triples; reason reasons over anything in the store. "ticket" is a domain, not a capability — it fails the test the other 43 pass.

And to your question — it is not "for tickets." It reads whatever IDS is loaded and returns the program + nearest precedents for a reference. Tickets are just today's data. It's a conduit from a reference to what memory already holds. That's a capability the list is missing: there's learn (write to memory) and understand (decode), but nothing named for the read-back — pulling the stored program + its precedents.

Recommendation: it's an intelligence skill, named for the capability, domain-free. The gap in the list is the read-back of memory, so: recall — given a reference, recall its program + the nearest precedents from the loaded IDS. It pairs with learn (learn writes, recall reads) and sits beside understand / reason. Alternatives if you prefer: retrieve (the plainer L2 term) or lookup. Not ticket, not mendo — the conduit is generic; the Mendo IDS is just what's mounted. New line in the intelligence list: intelligence/skill/recall.
