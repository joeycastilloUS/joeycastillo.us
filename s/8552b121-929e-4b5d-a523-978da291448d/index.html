Patterns applied — this is a Trust + Slot build, not new invention

The jacket is the Trust pattern (kastil-systems/a-d-d/patterns/trust.md) — Hollow Edge / Sealed Core / Anchor Seal — whose first known use is exactly our shape: switchboard (C# edge ▸ wire ▸ prime skill). The reusable C# artifact is the switchboard Hollow Edge at C:\switchboard\edge\Program.cs (155 lines, live, externally verified): CORS allowlist, Forward() with a scoped bearer, 429 retry, health, plus the Cloud Run scaffold. It's a hollow HTTP edge, not a wire client — by design.

The wire hop is already-written C: C:\nous\comms\skill\relay\relay_server.c has the edge-sink seam (POST /ingest, ~line 1995) — it takes {action, op, body} JSON from a C# Hollow Edge, strips it to wire triples, and delegates to the prime via relay_wire_to_prime(). Two production seams already ride it: airbyte-atlas/edge and yosemite-camp/conditions-edge.

The ticket→payload op is a Slot skill (slot.md): a self-contained seat at C:\nous\comms\skill\ticket\ (.c/.h/skill.json) + one Pin line in g_action_table (prime_server.c:6445). Verified from code: {"ingest",wire_h_ingest} + {"conditions",wire_h_conditions} pinned at 6460–6461. The ticket skill is the fourth rider on a proven bus — addition, not surgery, on the 6,995-line server.
