~d: collapse THINKING + MAIN AI — one concept, one control

Joey: "thinking and main ai are the same thing." Right. Today: a section labeled THINKING (live trace pane) + a labeled dropdown MAIN AI (picker) sit next to each other like two parts. They're not. They're one concept (the AI that answers) shown two ways (picker + trace). Merge.

1. Subject + Intent

Subject: the leftmost column of the cockpit — currently "THINKING" header + 2 stacked dropdowns ("MAIN AI" + "SUMMARY AI") + the live-trace body pane.

Intent: the operator sees ONE concept per row: this AI is doing this job, you can change it here. Stop labeling the column AND the picker — they describe the same thing.

2. Discovery

Two prior fixes shipped this session put labels here:

  • ~f 03:40 renamed DRIVER → MAIN AI, ENGINE → SUMMARY AI with subtitles. Better than before, but still treats the column header + picker label as two things.
  • ~f 04:00 added name parity (tool + model in both tile + dropdown). Helped but didn't address the section-header redundancy.

Joey's instinct on glance: he's reading "THINKING" and "MAIN AI" as two separate panes/concepts. They're not — they're a label and its input bound together.

Key shifts:

  • The header is GONE. No separate "THINKING" / "MAIN AI" pair.
  • Two rows: verb-as-label + dropdown. The verb describes what the AI does; the dropdown shows AND changes the AI. No redundancy.
  • The active-state badge is gone — the dropdown's current value IS the active state. Reading "THINKING: claude-opus — claude-opus-4-7" tells you everything.
  • Below: the live-trace pane (unchanged behaviorally; cleaner empty-state).
