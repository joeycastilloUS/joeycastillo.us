~d: DRIVER dropdown — claude-opus-4-7 default, NOUS-decide fail-silent

Two dropdowns in the THINKING header (DRIVER + ENGINE). Active state visible at all times. claude-opus-4-7 default for both. "Let NOUS decide" option fails silent when mesh is down. SENSE/SELECT trace stays — honest about what's happening.

1. Subject + Intent

Subject: A new "DRIVER" dropdown in the THINKING column header that controls which provider orchestrates the main task. A second labeled "ENGINE" stays for /summarize and /advise. Both default to claude-opus-4-7. Active state shown as a visible status line. When DRIVER is "Let NOUS decide" and the mesh isn't doing anything, the THINKING chain fails silent (no fallback driver).

Intent: The current cockpit lies. Dropdown says claude-opus-4-7; the trace says gemini. The orchestrator is whoever's first in g_defaults[], regardless of operator preference. Fix the lie by making the dropdown the source of truth, default it to Claude (Joey's pick for now), and surface the active driver visibly so the trace and the UI never disagree.

2. Discovery

Current landscape (from ~i 2026-05-26 00:10):

  • engine-select dropdown exists, defaults to "Let NOUS decide" (empty value)
  • JS sends set_engine WS message on change — server has no handler for it (silent drop)
  • handle_task picks orchestrator via first-available in g_defaults order; today = gemini
  • SENSE/SELECT trace strings are hardcoded literals in guided_ai_thread()
  • No failover on driver error

Gut lean: Joey wants Claude as the driver, wants to see what's actually driving, and doesn't want NOUS-decide to silently fall back to gemini. "Show me, don't tell me" applies — the trace must reflect reality.

JS

  • Both selects populated with: "Let NOUS decide" (value=""), then every provider name (value="claude-opus", label="claude-opus-4-7" etc.)
  • Default selectedDriver = "claude-opus", selectedEngine = "claude-opus"
  • On change: send WS {type:"set_driver", driver:"claude-opus"} or {type:"set_engine", engine:"claude-opus"}
  • Update #driver-active text to show current model name on every change
  • buildEngineSelector() grows into buildEngineSelectors() — handles both

C backend

  • New globals: char g_active_driver[64], char g_active_engine[64]; init to "claude-opus"
  • New WS message handlers: set_driver + set_engine (already-sent-but-dropped); both just strncpy into the globals
  • handle_task: instead of "first available," look up g_active_driver in g_providers; if found + has key, use that; if g_active_driver == "" (NOUS decide), check mesh availability, emit honest SENSE/SELECT trace, no fallback
  • SENSE/SELECT trace remains visible — but reflects actual state, not hardcoded gemini

Out of scope

  • Per-connection driver state (single-operator v1; one global suffices)
  • Real domain classifier ("pillar: GENERAL / 0.50" stays as a placeholder until NOUS sense is real)
  • Failover when a chosen driver errors (Joey's "fail and do nothing of course" applies — emit error, no auto-retry)
  • Persisting selected driver across sessions (in-memory only for v1)
