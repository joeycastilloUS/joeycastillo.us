New format type: openai-responses

Three additions to serve.c for the OpenAI Responses API (/v1/responses):

  1. New body builder build_openai_responses_body() — uses input field (single string, not messages array) and max_output_tokens (not max_completion_tokens). Hint is prepended to input.
  2. Response parser in extract_text() — output[0].content[0].text shape; skip past "type":"text" to the text-value key (same pattern as Anthropic content parsing).
  3. Usage parser in extract_usage() — uses input_tokens/output_tokens (same naming as Anthropic).
  4. Dispatch in the ctx->task body builder — new else if (format == "openai-responses") branch ahead of the openai fallback.
  5. Headers + URL fall through to the openai-default path (Bearer auth, endpoint as-is) since no special handling needed.
