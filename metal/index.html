Named workflow macros with loops. Define a chain once, run it forever.

~P  ~E  /~B  ~T/  ~S  ~R

Commands outside / run once, left to right. Commands inside / / loop until the exit condition is met: board exhausted, tests pass, nothing to fix.

Definition of Done

Every tilde command that produces an artifact must pass all 7 items before the command is considered complete. Ship (~s) adds 4 more. The gate holds.

✓ Code committed — descriptive message, no uncommitted work
✓ Commit hash in telemetry — backfilled in be/telemetry.json
✓ Board updated — item moved to Done with hash (if applicable)
✓ HTML report saved — correct datetime filename in be/
✓ Index updated — entry in be/index.html
✓ Pushed to remote — work exists on origin, not just locally
✓ No prior work damaged — Done items untouched unless explicitly targeted
Ship (~s, ~r) adds:
✓ Chain complete — every step has telemetry with non-null commit
✓ Site in sync — joeycastillo.us/metal updated and pushed
✓ ~h in sync — help display matches current commands
✓ Revision current — all footers use post-commit rN

Definition of Ready: Before the first edit of a session, read the board state, check the last telemetry entry, and verify the source of truth. 3 file reads. Takes seconds. Prevents work loss.

How It Works

Every tilde command triggers a full chain reaction: do the work, review it, render an HTML report, update the artifact index, commit, push, launch the demo. One signal in. Structured output.

~L load → ~I inspect → ~P plan → ~E estimate → ~B build → ~T test → ~F fix → ~S ship → ~R retro

Plans become sized kanban boards (Fibonacci: 3 or 5 points per item, 13-21 per board). Boards drive builds. Builds produce reports. Reports track everything. Every command accepts three forms: ~l = ~L = ~load.

48 boards shipped. 680+ story points delivered. The system is the structure. The tool is replaceable.
