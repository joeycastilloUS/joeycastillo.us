GPG Setup

Signed commits. Verified on GitHub. One-time setup.
← metal
Why
GPG signing proves every commit came from you. GitHub shows a green "Verified" badge next to signed commits. Without it, anyone can commit as your name/email and there's no cryptographic proof it was actually you.
This page walks through generating a key, wiring it to git and GitHub, and verifying the whole chain works. Takes about 2 minutes.
Setup
Step 1
Generate a GPG key
Create a 4096-bit RSA key tied to your git email. The batch command runs non-interactively.
gpg --batch --gen-key ^ --%%no-protection ^ Key-Type: RSA ^ Key-Length: 4096 ^ Name-Real: Your Name ^ Name-Email: your@email.com ^ Expire-Date: 0 ^ %%commit
gpg --batch --gen-key <<'EOF' Key-Type: RSA Key-Length: 4096 Subkey-Type: RSA Subkey-Length: 4096 Name-Real: Your Name Name-Email: your@email.com Expire-Date: 0 %no-protection %commit EOF
Replace Your Name and your@email.com with your git identity. The email must match what's in your git config.
Step 2
Get your key ID
List your secret keys and grab the 16-character key ID from the output.
gpg --list-secret-keys --keyid-format long your@email.com
Look for the line starting with sec rsa4096/ — the part after the slash is your key ID. Example: sec rsa4096/45992F4AD577C138 → key ID is 45992F4AD577C138
Step 3
Configure git to sign commits
Tell git which key to use and enable signing by default.
git config --global user.signingkey YOUR_KEY_ID git config --global commit.gpgsign true
Replace YOUR_KEY_ID with the 16-character ID from step 2.
Step 4
Grant GitHub CLI the GPG scope
The GitHub CLI needs the write:gpg_key scope to upload your public key. This opens a browser for OAuth approval.
gh auth refresh -h github.com -s write:gpg_key
This command is interactive. It will display a one-time device code and open your browser. Enter the code on GitHub to approve the scope. You only need to do this once.
Step 5
Upload your public key to GitHub
Export and pipe the public key directly to GitHub via the CLI.
gpg --armor --export your@email.com | gh gpg-key add -
gpg --armor --export your@email.com | gh gpg-key add -
If this succeeds, your key is now on GitHub. Every signed commit from this machine will show the green "Verified" badge.
Step 6
Verify it works
Make a test commit and check that it's signed.
git commit --allow-empty -m "test: verify GPG signing" git log --show-signature -1
You should see gpg: Good signature from "Your Name <your@email.com>" in the output. Push the commit and check GitHub — look for the green "Verified" badge.
Troubleshooting
gpg failed to sign the data
Usually means git is pointing to a key ID that doesn't exist on this machine. Run gpg --list-secret-keys --keyid-format long to see what keys you have, then update git config --global user.signingkey to match.
insufficient OAuth scopes
GitHub CLI doesn't have the GPG scope yet. Run step 4 — it's a one-time browser approval. Must be run interactively (not from a script or agent).
Commits show "Unverified" on GitHub
The email in your GPG key must match a verified email on your GitHub account. Check with gpg --list-keys your@email.com and compare against GitHub → Settings → Emails.
New machine, same identity
You can either generate a new key on each machine (and add each to GitHub), or export your private key and import it on the new machine:
# Export (on old machine) gpg --armor --export-secret-keys your@email.com > private.key # Import (on new machine) gpg --import private.key rm private.key
The private key file is sensitive. Transfer it securely and delete it immediately after import.
