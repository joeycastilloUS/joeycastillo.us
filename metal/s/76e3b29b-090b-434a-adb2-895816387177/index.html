~d~b — Per-Section Latchkey Widget + Personal URL

From Joey's comment on the last artifact ("this comments is not right, look at argus") — rebuild the Latchkey comment widget the argus way: per-section, with a Share button, on the promised personal cap-URL. Comment on any section above to try it.

Need

metal's Latchkey thread was one box at the bottom with a manual "section" field, so every comment landed as general — exactly what happened to Joey's feedback. argus (on neo) does it right: every section is commentable inline, and obj is set to that section automatically (surface / value / design / estimate…). And the ~i before this flagged the cap-URL as not-personal (hardcoded joeycastillo.us). This build closes both.

Design (the shape)

  • Per-section anchors. After every real <h2>, inject an empty <div class="lk-sec" data-obj="KEY">. KEY = the heading's id if it has one, else a stable slug of its text. One shared script fills each anchor with a clickable "N comments / Comment on this section" bar that expands an inline thread + form, posting with obj = KEY.
  • Whole-artifact box stays at the bottom (obj = page) for comments that aren't about one section.
  • Share button in the nav — copies the cap-URL (unchanged, kept prominent).
  • Personal cap-URL. PUBLIC_BASE / SITE_ROOT / prefix now come from be/config.json (latchkey section) or env — so each operator's Share button copies their own domain. Scotty sets public_base: https://scottyledell.us and his Latchkeys land there.
  • doc = token unchanged — so ~fu still resolves every comment (section or page) regardless of domain.
