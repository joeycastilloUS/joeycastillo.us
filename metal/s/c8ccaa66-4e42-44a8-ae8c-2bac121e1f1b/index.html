Build — Watch tab

Board WATCH1 COMPLETE. The llm-free connectivity watcher now has a face in the console: a live Watch tab that makes device silence visible. LIVE at argus.com.ai/app.

What Was Added

A read-only Watch tab in the Argus (meraki) console that fetches gcp-bridge /watch/state and renders one card per watched device:

  • Color-coded verdict badge — OK / FLAPPING / OFFLINE
  • flaps, offlineMin, status, humanized lastSeen + checkedAt
  • Staleness banner (>15 min since last check) — the tab's whole point: a stalled scheduler shows as stale, not as false calm
  • 45s auto-refresh while the tab is visible; empty + fetch-error states; read-only GETs only

Built on the existing GBRIDGE const and modeled byte-for-byte on the Templates tab pattern. The /watch/state endpoint is open (no auth); no write path anywhere.

Operator flag (Rule 13): the live /watch/state shows checkedAt ~37h old (last check 2026-07-08 03:25Z). The Watch tab correctly renders this as STALE. That staleness suggests the Cloud Scheduler marina-handheld-watch may have stopped firing — worth a look (the tab just made it visible, which is exactly its job). Separate from this build.

Decisions

  • Deliberate exception to the console's “fetch on click only” rule — a live monitor must auto-refresh; commented as such.
  • No auth header on the open endpoint (also avoids a needless CORS preflight).
  • Strings coerced (the endpoint returns flaps/offlineMin as strings).
