Intent

Bring metal.de (Desktop Edition) to full parity with metal.ce (Console Edition): the complete tilde contract — verbs, chains, board discipline, DoD/DoR gates, HTML report chain, telemetry, board-status, Signals, FQDN identity, ship/verify — executed natively on the operator's desktop, deterministic C at $0.00 where the work is deterministic (Tier 0), escalating to the Pantheon per tier where it isn't. Built in the fresh consolidated kastil-systems/metal-desktop repo (Joey's locked decision), which absorbs the biRT + Tier-0 engine from C:\tools\metal-de (r257) and the rebrand shell + 22-provider cockpit from C:\metal-desktop (r9). Built from a seeded a-d-d spec, never from context. Companion design: be/designs/2026-07-02-1117-design-metal-de-ce-parity.html.

Approach

  1. Spec first, code second. The a-d-d spec (*.proposed) and the origin re-verification land before any product commit. Build from a-d-d, never from context; drift stops work and flows up (Rules 10/11).
  2. Absorb, don't rewrite. Both trees merge with history so revision counts and via <hash> board references stay honest. The provider engine, WS broadcast, biRT engine, and sdk are lifted intact; the only new C is the seam (route_tilde) and the four organs (board_status / report_emit / telemetry / signal_emit).
  3. Prove the smallest seam, then widen by dependency ring. ~h end-to-end (~40 LOC per the 2026-06-12 inspect) proves frame → dispatcher → tier → round-trip. Then: local tildes → be/ machinery → report chain → Signals (queue) → identity → cross-platform → mesh → tlt → gate.
  4. Kernel owns semantics; DE routes and executes. Verb table lazy-loads from be/docs/commands.md via a configured path; slash and tilde stay separate namespaces; route / execute / report stay separated (tlt four-purpose discipline).
  5. Every item is a verify loop (~a~t / ~u~t / ~f~t, Rule 7); every phase exits through its own DoD including, where a deployable artifact exists, external verification (Rule 9). Signals and board-status are wired from the first tilde, not bolted on at the end — emission is the contract, not a feature.
  6. Plans are wrong by default. This is a v1 hypothesis; Unknowns and When To Revisit are first-class below. Phase 0's origin re-read is expected to move the target.

Constraints the 16-item checklist

  1. C99 everywhere we own; C# only as foreign-API jackets (Language Boundary lock 2026-06-20). The Playwright worker is the sole sanctioned jacket.
  2. No external deps — roll your own. libc + OS primitives; no OpenSSL, no frameworks. (Vendored Python renderer is named, temporary, contract-bounded debt.)
  3. a-d-d spec seeded first as *.proposed + ARCHITECTURE.md pointer; human approval before code; build from a-d-d never from context; drift stops work and flows up.
  4. Two-tree split resolved: fresh consolidated kastil-systems/metal-desktop (Joey, locked). Both sources absorbed with history; old homes frozen with MOVED READMEs.
  5. Kernel owns verb semantics; DE routes + executes. Lazy-load be/docs; never fork the tilde table into C; slash ≠ tilde.
  6. Tiered execution is the parity mechanism. Tier 0 pure C $0.00; tier: N in .pi frontmatter; escalation per tier; learned routing Cold/Warming/Hot.
  7. Wire protocol only into Purcius (NTRP0001, wire over TLS); never HTTP/REST/SDK into the substrate; new substrate capability = pure-C skill via Slot; secrets follow Trust. Foreign-AI fan-out stays direct-HTTPS (foreign edge, not substrate).
  8. FQDN = operator identity. CF verify + TOTP, no passwords, no third-party IdP; one root Ed25519 key never leaves the machine; <name>US/vault; _purcius.<fqdn> TXT.
  9. Signals are mandatory, not an edition feature. Every tilde emits one sealed Signal; local queue on mesh failure; local telemetry always; board-status.json schema v1 atomic writes; auto-board pop on ~b/~l.
  10. Full Be chain reaction per tilde: report → save → index → telemetry → commit (hash backfill) → push → DoD verify incl. external production verification → pop with pop-and-link.
  11. Cross-Platform From Day One: Win+Mac+Linux compile + smoke in the same ship cycle; named compat wrappers, no #ifdef spaghetti; MinGW UCRT (C:\tools\mingw64) is the WDAC-trusted Windows path; handlers designed for WDAC hand-off (interactive-terminal launch), never silent local exec under automation.
  12. Board discipline: triples {what, verb, context}, Fibonacci pts, verify-loop pairing (~a~t), Kaizen K-board, external-contributor via <hash> refs, historical be/ artifacts frozen.
  13. Versioning: v0 · rN (= commit count) · date · pts. No semver. No co-author lines, ever.
  14. Trade-secret hygiene: confidentiality header first line of every source file; NOTICE at root; env-var names only, never token values; commercial intel stays out of versioned files.
  15. Plan shape honored: Intent / Scope / Approach / Phases / Constraints + Unknowns + When To Revisit; plans are wrong by default; this banner stays until Joey approves.
  16. Done = production, externally verified (Rule 9): committed/pushed/green is not done; a third party verifies with HTTP 200 / dig / a fresh-machine build before any Done is written.

  • Kernel-docs source for non-Joey operators — the configured-path answer assumes a sibling metal clone; unknown is the install-time provisioning story on a fresh operator's box.

When To Revisit the v2 triggers

  • P0's origin re-read changes the 13-point surface — any moved target (new DoD items, Signals emit-per-tilde shipped in CE, biRT landed in CE) re-baselines the phases: plan v2 immediately, before P1.
  • The ~h proof fails the seam — if route_tilde + tier-0 can't round-trip in roughly the promised ~40 LOC + dispatch, the execution model (not the phase list) is wrong: back to ~d.
  • Sealed-IDS v3.1 turns out incompatible with the queued seal format — P6 re-opens and queued Signals get a re-seal migration.
  • Prime deploys — P9 unblocks; resequence it forward of whatever is then in flight.
  • Any drift between the approved a-d-d spec and what implementation discovers — per Rule 11 that stops the tilde and flows up as a spec-update proposal; two such stops in one phase = plan v2.
  • Tier assignments prove wrong in use (a Tier-0 spec keeps escalating, or a Tier-2 tilde turns out deterministic) — retune frontmatter numbers per the tiered design's escape route; a pattern of retunes = revisit the tier table in the spec.
