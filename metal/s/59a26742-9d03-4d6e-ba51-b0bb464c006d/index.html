Retro — Latchkey + Comments + Worktree

Closing ~GO. The arc that made every metal artifact a shareable, commentable cap-URL and let one box run many sessions.

What Shipped

  • Latchkey — pattern (a-d-d), C99 skill spec (nous/comms), canonical engine (cap_tokens.py + latchkey_publish.py), and the Publish Protocol wired into the chain. Every artifact is born a cap-URL at /metal/s/<token>/, hardened + robots-disallowed.
  • Comment loop — a Thread widget + real Share button on every artifact, keyed on its token; ~fu is the on-demand Attendant (sweep → act on ~-commands → reply → mark seen). Proven live end-to-end.
  • Concurrent sessions — go <n> runs session n in its own git worktree; two sessions land on origin/main with zero index.lock contention.
  • Fleet-ready install — one-stop self-heal (archived-origin repoint fixed), MCP purged, framing corrected.

What Worked

  • Self-proving First Moves. The Latchkey design round-tripped a comment through its own mechanism; the worktree First Move was proven in an isolated harness before shipping. Proof over claim.
  • Reuse before invent. Lifted the proven mint/ledger/stage engine rather than rebuilding; the only inventions were placement, the widget, three ledger fields.
  • Fable on the a-d-d + designs. Architecture and design authorship stayed with Fable 5; the metal chain (commit/publish/push) stayed with the main session. Clean division.
  • Dogfooding. The build, inspect, ship, and this retro are all Latchkeys — the capability is exercised by shipping it.
