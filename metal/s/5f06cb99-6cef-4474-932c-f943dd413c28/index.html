Design — metal.de ↔ metal.ce Parity foundational

The Desktop Edition executes the full tilde contract natively — deterministic C at $0.00 where the work is deterministic, escalating to the Pantheon per tier — inside one fresh consolidated repo that absorbs both existing DE trees.

DECISION (locked by Joey): the canonical DE is a FRESH CONSOLIDATED repo — kastil-systems/metal-desktop — absorbing (a) the biRT + Tier-0 pure-C engine from C:\tools\metal-de (kastil-systems/tools, r257) and (b) the clean rebrand shell from C:\metal-desktop (joeycastilloUS/metal-desktop, r9). This design designs against that.

1. Need

The software factory that builds itself cannot run on a per-command LLM bill. CE proves the discipline works — 1468 revisions, 200 builds, 91 ships — but every tilde routes through Claude Code, so every deterministic step (re-reading the same recipe, walking the same 9-step skeleton) costs tokens and seconds. The receipt is exact: an LLM-driven ~unload ran 47.9s / $0.146 and didn't finish; the pure-C Tier-0 version runs ~100ms / $0.00 and does. The 17-run analysis showed 100% of ~load runs re-discovering the same static recipe. LLM is the exception, not the default — and the desktop is where the operator's learned .pi library pays off. What's missing is one canonical Desktop Edition that embodies the Be kernel natively: same tilde vocabulary, same chain reaction, same board/DoD/Signals contracts — executed by deterministic C, escalating to the Pantheon only where synthesis is genuinely needed. Today that capability is split across two half-built trees (the engine without the home, the home without the engine), and neither emits a Signal, writes board-status, or renders a report. Parity means DE becomes a full citizen of the discipline, not a sibling cockpit that watches it.

2. Raw User Stories

  • As the operator, I want ~h ~l ~b ~ul to execute on my desktop in ~100ms at $0.00, so the factory doesn't bill me for work that never changes.
  • As the operator, I want the exact same tilde vocabulary in DE as in CE (three forms, chains, verb semantics), so I never learn two systems and the structure never drifts.
  • As the operator, I want every DE tilde to write board-status.json, log telemetry, render a Be report, and emit a sealed Signal, so the live board and the Pantheon see DE work exactly as they see CE work.
  • As Joey, I want one canonical DE repo under kastil-systems with both histories absorbed, so the two-tree confusion ends today and every via <hash> board reference still resolves.
  • As a future operator on Mac or Linux, I want the cockpit, fan-out, and tilde engine to compile and run off-Windows, so DE honors Cross-Platform From Day One instead of hiding behind a WinHTTP stub.
  • As the kernel steward, I want DE to read be/docs/commands.md as the verb authority and route mechanically, so the tilde table is never forked into C.
  • As Joey, I want the architecture seeded in kastil-systems/a-d-d/_projects/metal-desktop/ before code lands, so DE is built from spec, not from context.

5. Shape

Minimum viable structure — reuse the router seam, the provider engine, and the WS broadcast; build the tilde organs around them:

  • The repo skeleton — src/ (serve.c base + graft), src/birt/ (absorbed engine), sdk/ (wire/crypt/http), wwwroot/ (cockpit), be/ (full CE-shape ledger: docs-pointer, board.md, board-status.json, builds/plans/designs/tests/ships/retros, tools/), ARCHITECTURE.md pointer, NOTICE, trade-secret headers, version_gen.h stamping, one Makefile + make.bat.
  • route_tilde() — the one new branch at serve.c:3056. Reads the tilde frame, resolves the verb against the kernel-owned spec (lazy-loaded be/docs/commands.md, path in desktop.conf; never a forked table in C), hands to birt_dispatch(). Front-end: loop.js sends a tilde frame when input starts with ~; slash commands stay a separate namespace.
  • The Tiered Execution Chain (absorbed) — birt_dispatch reads tier: N from the .pi spec frontmatter; Tier 0 walks the JSON ops protocol via birt_fs + spawn_compat ($0.00); Tier 1 adds per-op escalation; Tiers 2/3 run the existing birt_loop with spec primer, budget rail, driver routing, and trace. Multi-tilde chaining per the tiered design (context + args).
  • The four organs — board_status.c (atomic status writes start/phase/end, drives the live spinner), report_emit.c (Be-template HTML + DoD checklist + index update + pop-and-link), telemetry writer (telemetry.json 100-ring, commit-hash backfill), signal_emit.c (one sealed Signal per tilde; local queue when no relay; local telemetry always written — Signals are not an edition feature, they're the contract).
  • Vendored view layer — render-board-view.py + watch.py + open-board, driven purely through the board.md / board-status.json contract, so the C-native renderer can replace it without touching any writer.
  • Identity + auth — un-bypass the auth gate (one-line initAuthGate revert), FQDN handshake + CF token verify + TOTP composing from crypt.c, the 5 dormant relay flows refactored onto relay_client.c. Trust-shaped: hollow edge, sealed core, keys never leave the machine.
  • Cross-platform floor — http_compat fills the Linux/Mac HTTPS stub; judge threading moves behind the existing thread pattern instead of #ifdef _WIN32-only; Win+Mac+Linux compile+smoke in the same ship cycle becomes a DoD gate.

Explicitly OUT of parity v1:

  • The mesh leg of prime-dependent tildes (~s Store publish, Signal drain, intelligence loop) — queued locally until a relay+prime is deployed (that stand-up is external to this repo). No-prime v1 is legitimate; silent failure is not — queue and say so.
  • C-native board renderer (v1 vendors Python behind the contract), terminal tab / /ws/terminal, xterm.js replacement.
  • ALF fold (master P9), biRT v3 (P13), 38 driver manifests (P14), IDS self-storage (P12) — master-plan far horizon, not parity.
  • MCP (currently-purged posture in CE; DE ships none).
  • Backlogged Chinese-provider cohort; new cockpit paradigms; silicon console.

6. First Move

Seed the architecture: write kastil-systems/a-d-d/_projects/metal-desktop/ as *.proposed — the DE spec (intent, parts, the four contracts: tilde frame schema, tier model, board-status/Signal/report shapes, consolidation map) — and put it in front of Joey for approval. Zero product code before the spec exists.

Why this and not the ~40-LOC ~h proof: the ~h round-trip is the smallest code proof and it stays exactly where the inspect put it — but it becomes Phase 2's exit criterion, not the First Move. This design exists to end drift between two trees; building even 40 lines from context before the spec exists would violate the three rules that make DE trustworthy: Stewardship (a-d-d is human-owned, Claude proposes, Joey approves), flow-up-before-down (architecture → tests → code), and build-from-a-d-d-never-from-context. The factory's whole pitch is that the structure never drifts; the keystone project doesn't get to skip the keystone rule. The spec is also cheap — this design and the three deep-read reports are 90% of its content.

These go into the plan: consolidation (subtree-split + history merge + MOVED freezes), the ~h native proof through route_tilde(), local no-prime tildes, the four organs + vendored view layer, the report chain, Signals local-queue, identity/auth, cross-platform floor, relay-dependent tildes, tlt/~t binding, and the full-parity gate with external production verification. Sequenced in the companion ~p.
