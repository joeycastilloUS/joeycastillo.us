Ship — Latchkey Comments Are Default in metal.CE

Every artifact metal.CE makes is now born a Latchkey with the canonical per-section comment widget, a Share button, and a personal cap-URL — no extra step. This is the default. Comment on any section below to see it.

What's Default Now

  • Per-section comments (the "kung ~fu" template). Every <h2> section gets a subtle "comment" link, a collapsible heading, an N-new badge, and a live dot — the exact artifact-comments.js Joey blessed on mendo-zendesk-ai. Hosted once at /metal/artifact-comments.js.
  • Share button in the nav — copies the cap-URL.
  • Personal cap-URL. PUBLIC_BASE / SITE_ROOT come from be/config.json (latchkey section) — each operator publishes to their own domain. Scotty → scottyledell.us.
  • doc = the cap-token — so ~fu resolves every comment (section or page) regardless of domain.

Made Default

  • latchkey_publish.py marks the first block after each <h2> (div / section / ul / ol / table / pre / blockquote) with data-cm, injects window.ARTIFACT={doc:token} + the shared script. Idempotent.
  • be/tools/artifact-comments.js is metal's canonical widget (verbatim from Joey's template); hosted on the site at /metal/artifact-comments.js.
  • be/config.json carries the latchkey section (personal base/root/prefix).
  • Canon updated: be/docs/conventions.md § Latchkey Publish Protocol now describes the per-section widget + personal URL as the default. The chain reaction already runs latchkey_publish on every artifact — so this is automatic going forward.
  • All artifacts re-published on the new widget.
