Retro — F5 Hardening + Fleet Install

Deep-analyze the tool surface, fix what a fleet reinstall would hit, then meet reality on real machines.

What Shipped

  • 4-agent deep code analysis (~50 findings) → fixed the data-loss, XSS, Mac/Linux-commit, voice-prompt, telemetry, and launch classes.
  • Live-install hotfixes: Go.Sign noreply email, install force-refresh of Go.Fe/Go.Sign, site sync, MCP keep-list + backup, per-operator repo access.

What Worked

  • Parallel deep analysis caught real blockers the earlier "GO" passes missed — autoheal silently eating uncommitted work, and board text reaching published pages as live HTML. Static analysis before the fleet reinstall was the right order.
  • Harness-verifying the risky fix. Autoheal runs on every launch; the data-loss fix was proven in a 4-case git harness (preserve / reconcile+restore / ff / skip) before shipping. Proof over hope.
  • Reality found what analysis couldn't. Running the install on real machines surfaced the stale-clone trap, the Go.Sign scope failure, the access wall, and the MCP-torch — none visible from reading code on one box. Ship, watch, fix.
